The following documentation is related to Vulnerabilities:

Remediating Ubuntu vulnerabilities

Source: concert-2.2.x-documentation.md (concert) Use the Ubuntu patching capability to automatically remediate vulnerabilities identified in Ubuntu virtual machines (VMs). When you upload an Ubuntu vulnerability scan file, Concert retrieves advisory details from the official Ubuntu CVE feed, generates remediation actions for affected packages, and applies package upgrades through the auto-remediation workflows.

Figure 1. Determining how vulnerabilities are prioritized

Source: concert-2.2.x-documentation.md (concert) Details of vulnerability prioritization options The following are the details of the vulnerability prioritization options: Concert risk score: Uses severity (CVSS), exploitability (EPSS), and environmental factors to determine the risk of each CVE instance. For details about how the Concert risk score is calculated, see Understanding the Concert risk score for vulnerabilities. Note: For CVEs without a CVSS score, Concert assigns a default Severity component based on the CVE’s lastModified date. CVEs published or modified within the last 60 days are assigned a value of 10, while older CVEs are assigned 5. This ensures newly published unscored CVEs are treated with caution. Common Vulnerability Scoring System (CVSS): An industry standard that measures the intrinsic characteristics of a vulnerability that do not change across environments. External score through API: A user-defined value submitted through the API.
