Applying tags

Source: concert-2.2.x-documentation.md (concert) You can apply tags to inventory objects, such as applications, environments, or individual components, to help with categorization and filtering.

Click Save to apply the tags. To cancel, select Cancel.

Source: concert-2.2.x-documentation.md (concert) Assigned tags will appear in the Tags column of the applications row. Importing data to Concert There are multiple methods you can use to import application and dimensional data to your Concert instance. Supported data ingestion methods Concert toolkit Using the Concert toolkit, you can automate data ingestion as part of your CI/CD pipeline. This is the recommended approach as it helps ensure that your application and environment data stays up to date. Concert API The Concert API includes a data ingestion endpoints that you can use to upload SBOM files, scans, and other data types. View API documentation. Note: When uploading scan data by using the Concert API, include the optional scanner_name parameter in your request. Concert uses this value to track the scan source and archive outdated findings when a new scan is uploaded for the same image.

Optional: Add tags to your workflow metadata.

Source: concert-2.2.x-documentation.md (concert) a. Expand the Workflow tags accordion. b. In the Values field, enter meaningful tag values. For example, for the Category tag, enter values that indicate the purpose of your workflow, such as Remediation. If you intend to make this workflow externally available by exposing it, these tags can then be used by external systems to identify, categorize, or filter the workflow.
