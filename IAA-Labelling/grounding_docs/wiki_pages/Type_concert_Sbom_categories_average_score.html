Refreshing a package SBOM (CycloneDX)

Source: concert-2.2.x-documentation.md (concert) The SBOM package that you upload has details such as OpenSSF scorecard, Recommendations, File view, History, and others. But the score is generated at the time that the package is added and is not dynamically refreshed. Now, you can refresh your package details on demand with Concert.

Understanding reliability score

Source: concert-2.2.x-documentation.md (concert) Once you upload a package SBOM file in CycloneDX format, Concert uses the Open Source Security Foundation (OpenSSF) scorecard tool to assess the security posture of the open-source software packages in your applications and generate a package reliability score based on the criticality of a package-related issue.
