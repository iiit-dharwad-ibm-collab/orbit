Fields

FieldData TypeListDescription
beam_widthIntSpecifies the number of beams that are used in beam search decoding. A higher value allows for a more exhaustive search, potentially improving the quality of the output at the cost of computational complexity.
decoding_methodStringRepresents the strategy that is used for picking the tokens during generation of the output text. Possible values include 'greedy' for deterministic generation and 'sample' for stochastic generation.
include_stop_sequenceBooleanPass `false` to omit matched stop sequences from the end of the output text. The default is `true`, meaning that the output ends with the stop sequence text when matched.
length_penaltyLengthPenaltyInputIt can be used to exponentially increase the likelihood of the text generation terminating once a specified number of tokens have been generated.
max_new_tokensIntThe maximum number of new tokens to be generated. It limits the length of the generated output, preventing it from exceeding a specified size.
min_new_tokensIntIf stop sequences are given, they are ignored until minimum tokens are generated.
random_seedIntRandom number generator seed to use in sampling mode for experimental repeatability.
repetition_penaltyFloatRepresents the penalty for penalizing tokens that have already been generated or belong to the context. The value 1.0 means that there is no penalty.
return_optionsReturnOptionsInput
stop_sequencesStringYesStop sequences are one or more strings which will cause the text generation to stop if/when they are produced as part of the output.
temperatureFloatA value used to modify the next-token probabilities in sampling mode. Values less than 1.0 sharpen the probability distribution, resulting in \"less random\" output. Values greater than 1.0 flatten the probability distribution, resulting in \"more random\" output. A value of 1.0 has no effect.
time_limitIntTime limit in milliseconds - if not completed within this time, generation stops. The text generated so far is returned along with the TIME_LIMIT stop reason.
top_kIntThe number of highest probability vocabulary tokens to keep for top-k-filtering. Only applies for sampling mode. When decoding_strategy is set to sample, only the top_k most likely tokens are considered as candidates for the next generated token.
top_pFloatSimilar to top_k except the candidates to generate the next token are the most likely tokens with probabilities that add up to at least top_p. Also known as nucleus sampling. A value of 1.0 is equivalent to disabled.
truncate_input_tokensIntTruncates the input to a specified number of tokens to prevent exceeding token limits, which may cause request failures.
typical_pFloatMeasures typicality to control how predictable the next token is compared to the expected randomness. Lower values lead to more diverse outputs.
