Transaction log resize actions

Source: IBMTurbonomic_8.16.0.md (turbonomic) Resize actions based on the transaction log resource depend on support for virtual storage in the underlying hypervisor technology. Currently, Turbonomic does not support resize actions for Oracle and Database Servers on the Hyper-V platform (due to the lack of API support for virtual storage).
