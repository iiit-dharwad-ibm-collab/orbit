Discovering images in an image registry

Source: concert-2.2.x-documentation.md (concert) Using Concert Workflows, you can configure Concert to auto-discover images that are present in an image registry on a virtual machine (VM). Once ingested to Concert, the workflow scans and inspects the images to compute resilience metrics related to the image layers and size. Running image scans for resilience proactively finds hidden vulnerabilities to prevent breaches, reduce risk, and maintain system integrity, confirming that images are secure, reliable, and meet industry benchmarks. Before you begin The application corresponding to the images must be defined in your Concert inventory. Step 1: Download and import the prebuilt workflow In Concert Workflows, in the Workflows page, click Create workflow > Select from library, choose the Docker_Image_Registry_Workflow and import the workflow to your Concert Workflows instance. For more information about importing Concert-specific workflows into Concert Workflows, see Creating workflows.
